# CAD-приложение Qt_OCC

Qt_OCC — это настольное CAD-приложение, разработанное с использованием возможностей фреймворка Qt для пользовательского интерфейса и OpenCASCADE Technology (OCCT) в качестве геометрического ядра. Оно призвано предоставить платформу для 3D-моделирования и визуализации, объединяя в себе ряд функций для создания форм, манипулирования ими и анализа.

## ✨ Возможности

### 📂 Управление файлами
*   **Ввод/Вывод**: Поддержка создания новых проектов, открытия существующих файлов, сохранения и экспорта моделей.

### 🎥 Визуализация и Навигация
*   **Стандартные виды**: Быстрое переключение между изометрическим видом, видами сверху, снизу, слева, справа, спереди и сзади.
*   **Режимы отображения**: Переключение между каркасным режимом (Wireframe), затенением (Shaded) и затенением с отображением ребер (Shaded with Edges).
*   **Управление видом**: Автоматическая подгонка вида под содержимое сцены.

### 🛠️ Инструменты моделирования
*   **2D Примитивы**: Создание точек, линий, прямоугольников, кругов, дуг, эллипсов, многоугольников, кривых Безье и NURBS-кривых.
*   **3D Примитивы**: Генерация параллелепипедов (Box), сфер, цилиндров, конусов и пирамид.
*   **Булевы операции**: Выполнение операций объединения (Fuse), пересечения (Common) и вычитания (Cut) над твердыми телами.
*   **Модификации**:
    *   **Отражение**: Зеркальное отражение объектов относительно плоскости или оси.
    *   **Массив**: Создание линейных и круговых массивов.
    *   **Оболочка (Shell)**: Преобразование твердых тел в оболочечные конструкции.

### 📏 Измерения и Анализ
*   **Измерения**: Инструменты для точного измерения расстояния, длины ребра, длины дуги/радиуса и углов.
*   **Анализ**: Проверка на пересечения (интерференцию) между объектами.
*   **Визуализация сборки**: Разнесенный вид для осмотра внутренних компонентов.
*   **Сечения**: Применение секущих плоскостей для просмотра поперечных сечений.

### ⚙️ Утилиты и Настройка
*   **Трансформация**: Интерактивные инструменты для перемещения, вращения и масштабирования объектов.
*   **Рабочая плоскость**: Настраиваемый помощник системы координат.
*   **Фильтры выбора**: Фильтрация выбора по вершинам, ребрам, граням или твердым телам.
*   **Поддержка тем**: Переключение между светлой и темной темами оформления.
*   **Локализация**: Поддержка нескольких языков.

## 🛠️ Технологический стек

*   **C++**: Основная логика приложения написана на современном C++ (стандарт C++17).
*   **Qt Framework**: Используется для всего графического интерфейса пользователя (GUI), с применением SARibbon для современной ленточной панели инструментов.
*   **OpenCASCADE Technology (OCCT)**: Мощное геометрическое 3D-ядро с открытым исходным кодом, обрабатывающее все операции CAD.
*   **CMake**: Кроссплатформенная система сборки, используемая для настройки и сборки проекта.

## 🚀 Начало работы

Следуйте этим инструкциям, чтобы получить копию проекта и запустить ее на локальном компьютере для разработки и тестирования.

### Предварительные требования

Вам необходимо установить следующее программное обеспечение:

*   **Компилятор с поддержкой C++17**:
    *   Windows: MSVC 2019 или новее
*   **CMake**: Версия 3.16 или выше.
*   **Qt Framework**: Версия 6.2 или выше. Убедитесь, что установлены компоненты, соответствующие вашему компилятору.
*   **OpenCASCADE Technology (OCCT)**: Версия 7.6.0 или выше.

### Шаги сборки

1.  **Клонирование репозитория**

    ```bash
    git clone https://github.com/bezierC0/Qt_OCC.git
    cd Qt_OCC
    ```

2.  **Создание директории для сборки**

    ```bash
    mkdir build
    cd build
    ```

3.  **Настройка CMake**

    Убедитесь, что вы указали правильные пути к установкам Qt и OCCT.

    ```bash
    # Пример для Windows с Visual Studio
    cmake .. -G "Visual Studio 17 2019" -A x64 \
    -DQt5_DIR="C:/Qt/5.15.2/msvc2019_64/lib/cmake/Qt5" \
    -DOpenCASCADE_DIR="C:/Libs/OCCT/occt-7.8.0-vc14-64" \
    -DOpenCASCADE_3RDPARTY_DIR="C:/Libs/OCCT/3rdparty-vc14-64"
    ```

4.  **Сборка проекта**

    ```bash
    cmake --build . --config Release
    ```

5.  **Запуск приложения**

    Исполняемый файл будет находиться в директории `build/bin/Release` (на Windows) или `build/bin` (на Linux).
